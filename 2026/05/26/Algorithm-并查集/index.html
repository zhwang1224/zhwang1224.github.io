并查集

1. 通信录合并

描述

输入：
第一行表示通讯录的记录数量 num，值的范围[1,1000]。
从第二行开始，每一行代表一条联系人记录，每条录包括姓名和若干电话号码，电话号码的个数不超过10（姓名由英文字母大小写组成，长度在1到10之间，电话号码由数字组成）。
之后的若干条记录代表人的记录（手机号个数1到10位数字不重复）。同一条记录里的电话号码用空格隔开，电话号码的长度在[1,10]

输出：
整理后的通讯录列表，其中如果两个联系人被识别为同一个人，则他们的电话号码合并。
输出联系人姓名和手机号码列表按ASCII码升序排序。

具体思路

这题的核心思路是用 并查集 合并联系人。先把每条联系人记录看成一个独立集合，用一个哈希表记录“手机号第一次出现在哪条记录”。遍历每条记录的手机号时，如果手机号之前出现过，就说明当前记录和之前那条记录属于同一个人，用并查集合并它们；如果没出现过，就记录该手机号对应的当前记录编号。所有记录处理完后，再遍历一遍联系人，根据 find(i) 找到每条记录所属的根节点，把同一根节点下的姓名和手机号合并起来。姓名取字典序最小的，手机号用 set 去重并按 ASCII 升序保存。最后把每个合并后的联系人放入结果数组，整体排序后输出。

2. 通信录合并

本题给定一个 N×N 的相似度矩阵 M，M[i][j] 表示第 i 张和第 j 张图片的相似度，矩阵对称且取值范围为 0 到 100。当两张图片的相似度大于 0 时认为它们相似，并且相似关系具有传递性：若 A 与 B 相似、B 与 C 相似，即便 A 与 C 的相似度为 0，三者也会被归入同一个相似类（但 A、C 之间不额外计入相似度）；若某张图片与其余所有图片都不相似，则它自己单独成一类，相似度之和记为 0。最终需要计算每个相似类内部所有图片对的相似度之和，并按从大到小的顺序输出。剥开图片分类的外壳，这道题的本质其实是一道图的连通分量求和问题：把图片看作节点、相似度大于 0 的关系看作带权边，每个相似类就是一个连通分量，所求即各连通分量的边权总和。

具体思路

由于相似关系具有传递性，天然适合用并查集（DSU）来维护连通分量：遍历矩阵上三角，对每一对满足 M[i][j] > 0 的图片执行合并操作，处理完后属于同一类的图片就落在同一个根节点下。接着再次遍历所有图片对，对每条相似度大于 0 的边，把权值累加到其所在连通分量的根节点对应的累加器上（用哈希表以根节点为键存储），即可得到每个相似类的相似度之和。这里有一个容易被忽略的细节：与所有图片都不相似的孤立点也构成一个独立的相似类，其和为 0，必须保留在结果中，因此在收集答案时不能把值为 0 的类过滤掉。

3. 网络连接拒绝执行数

题目内容

有一些网络设备，其中某些设备具备互斥特性，这些设备中任意两台设备都无法直接或间接的连接在一起。 网络工程师按照给定的一系列指令将设备两两连接，可一旦网络工程师判

断连接指令触发了互斥规则，他将拒绝执行这条指令、将其记录并继续执行后续指令。

求所有指令执行完毕后，网络工程师拒绝执行指令的总数。

核心思路

建立并查集类，在类里面维护一个用于查询当前根节点是否拥有互斥设备的向量表。首先将输入的互斥设备序号用于向量表的初始化，然后每当执行指令时首先查询指令是否导致互斥，若互斥则不执行连接；其次若不导致互斥，则进行连接，然后统计不执行指令个数即可。
