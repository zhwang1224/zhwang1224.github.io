1. Agentic AI

普通的 LLM 应用通常是“ 用户提问 -> 模型回答 -> （循环）结束”的单轮生成模式，而 Agentic AI 更强调“用户提出目标 -> 系统拆解子任务 -> 调用工具/搜索/代码/数据库 -> 结果检查 -> 反思修正 -> 输出最终结果” 的多步骤（并可包含多 agent 并行执行）执行流程。

Agentic AI 的价值已经充分体现，它已经被用于企业客户支持、深度研究、法律文档处理、医疗辅助判断等复杂任务中。而真正有效的 Agent 应用，需要通过工作流设计、任务分解、工具调用、评估与误差分析，让大模型从单纯的“文本生成器”升级为能够持续行动、迭代改进并完成复杂任务的“任务执行系统”。

Agent 有不同的自主性程度，可以理解的是对于高自主度的 Agent 拥有更多的选择，甚至可以在执行过程中创建新的工具或流程；而对于低自主度的 Agent 而言，其执行步骤通常提前设置，其调用的工具也通常是硬编码，其自主性主要体现在 LLM 生成的文本内容。

Agentic workflows 的最大价值可能就是模块化（灵活），通过模块化的执行步骤可以达到比使用单独的 LLM 应用所得到的性能要好；可以并行执行人力要花巨大时间完成的复杂或工作量很大的任务。

课程总结了四个智能体工作流的设计模式：1. Reflection; 2. Tool use; 3. Planning; 4. Multi-agent collaboration。通过单个 LLM/agent 的迭代通常可以提高输出质量；LLM 可以调用越来越多的工具，通过多模态的输入来完成特定任务；不同任务的规划；多 Agent 的协作。

1.2 Tools

当我们询问大模型有关于当前时事热点的问题时，大模型的资料库中通常不会有最新的热点信息，但当我们允许大模型调用相应函数去获得这些信息时，这些信息会反馈给LLM，而此时大模型便可以给到我们想要的输出。即工具就是我们提供给大模型的函数，而函数可以请求调用。

在 LLM 原生不支持使用工具的情况下，若要使用 tools，则流程为 首先实现相应功能函数，并把函数提供给 LLM；当需要使用该工具时，LLM会生成一个特定输出，告知需要为LLM调用函数；调用后获取函数输出，并反馈给LLM，LLM会用这个结果继续执行后续操作。
