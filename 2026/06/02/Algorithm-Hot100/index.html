Hot100 算法题思路汇总

数组与哈希

  1. 两数之和

给定一个整数数组 nums 和一个整数目标值 target，请你在该数组中找出 和为目标值 target 的那 两个 整数，并返回它们的数组下标。

具体思路：创建哈希表存储整数值及对应序号，然后通过遍历一次数组，寻找每个当前值与其对应 match 值，返回结果。在 unordered_map 中，平均查找和插入都是 O(1)，时间复杂度：O(n)；空间复杂度：O(n)

  1. 字母异位词分组

给你一个字符串数组，请你将 字母异位词 组合在一起。可以按任意顺序返回结果列表。

思路：可将每种字符串按照升序重新排序，形成固定的字符串作为哈希表的 key。遍历每个字符串，如果是新类别则新创建分组，否则通过查表得到分组序号插入即可。时间复杂度：O(n · k log k) 空间复杂度：O(n · k)。

  1. 最长连续序列

给定一个未排序的整数数组 nums ，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。
请你设计并实现时间复杂度为 O(n) 的算法解决此问题。

思路：将整数数组整体放入哈希表中，然后遍历哈希表，对每个元素向上溯源直至找到第一个元素，然后计算这段序列长度。遍历后得到最长连续序列。平均时间复杂度：O(n) 空间复杂度：O(n)。

双指针方法

  1. 移动数组中的0
    给定一个数组 nums，编写一个函数将所有 0 移动到数组的末尾，同时保持非零元素的相对顺序。请注意 ，必须在不复制数组的情况下原地对数组进行操作。

思路：建立双指针，左指针指向下一个需要处理的0元素，右指针指向下一个需要处理的非零元素，遍历数组然后交换左右指针的元素即可。平均时间复杂度：O(n) 空间复杂度：O(1)。

  1. 盛最多水的容器

给定一个长度为 n 的整数数组 height 。有 n 条垂线，第 i 条线的两个端点是 (i, 0) 和 (i, height[i]) 。

找出其中的两条线，使得它们与 x 轴共同构成的容器可以容纳最多的水。返回容器可以储存的最大水量。说明：你不能倾斜容器。

具体思路：建立双指针，循环更新瓶颈指针，并计算每次的容纳水量，直至双指针相遇。时间复杂度：O(n) 空间复杂度：O(1)

  1. 三数之和

给你一个整数数组 nums ，判断是否存在三元组 [nums[i], nums[j], nums[k]] 满足 i != j、i != k 且 j != k ，同时还满足 nums[i] + nums[j] + nums[k] == 0 。请你返回所有和为 0 且不重复的三元组。

注意：答案中不可以包含重复的三元组。

具体思路：哈希表写法不如排序 + 双指针稳定、好去重。先对数组进行升序排序，然后遍历所有的数，并用双指针分别指向其下一个元素和数组最后一个元素，如果和为0则加入向量中，若和大于0则right–，若小于0则left++。

去重包括三部分，在外层遍历时若下一个元素等于当前元素则可跳过此次循环；进入循环后跳过重复的left和right。时间复杂度：O(n^2) 空间复杂度：O(1)

  1. 接雨水

给定 n 个非负整数表示每个宽度为 1 的柱子的高度图，计算按此排列的柱子，下雨之后能接多少雨水。

具体思路：每个位置能接多少水，取决于它左边最高柱子和右边最高柱子中较矮的那个。water[i] = min(leftMax[i], rightMax[i]) - height[i]。时间复杂度：O(n) 空间复杂度：O(1)
