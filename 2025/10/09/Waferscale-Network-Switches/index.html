晶圆级网络交换机 Waferscale Network Switches

文章指出尽管网络交换机的端口数（radix）在决定现代计算机系统的延迟、成本、功耗、空间和能力方面至关重要，但由于片外 I/O 间距和交换芯片尺寸扩展不佳，交换机端口数多年来几乎没有增长。文章考虑采用晶圆级集成（WSI）来将交换机基板的尺寸扩展到远大于单个芯片，并提出问题：能否利用 WSI 构建端口数远超当今交换机的网络交换机？

晶圆级网络交换机的端口数限制并非来自基板面积，而是受到 SSC 间连接的片内带宽所限制（第 IV 节）。为了支持 Clos 拓扑，尤其是在高层次上，需要大量逻辑连接，这会迅速耗尽相邻 SSC 间可用的物理带宽密度。文章还发现，当采用晶圆边缘基于 SerDes 的传统片外连接时，片外带宽也可能成为瓶颈。最后，功率密度也可能成为瓶颈，从而限制在给定散热机制下可支持的端口数。

1. 利用晶圆级集成构建超高端口数交换机

1.1 WSI

晶圆级集成 (WSI) 允许在整个硅晶圆尺度上构建集成电路。与传统方法不同，后者通过互连独立封装的芯片来构建大型系统，这些互连的带宽较低、能耗较高；而 WSI 则可以直接构建极大规模、紧密耦合的系统。WSI 技术可以分为两类：单片式 (monolithic) 和 芯粒式 (chiplet-based)。

  • 单片式 WSI：使用整片晶圆作为一个完整芯片，而不是切割成小芯片。上世纪 70 年代 Trilogy 系统架构和 2020 年的 Cerebras 晶圆级引擎 (WSE) 就是例子。在单片式 WSI 中，晶圆依然采用传统的步进曝光工艺制造，但不同之处在于不切割，而是在晶圆后续工序中构建跨视场（reticle）的互连。这样可以形成一个巨大的、紧密耦合的晶圆级系统。
    主要挑战包括：

    • (a) 制造过程存在缺陷和参数偏差，因此需要内建冗余来保证良率；
    • (b) 由于只能使用单一工艺节点，这种方式仅支持同构架构，无法在同一晶圆上整合不同工艺优化的芯粒（如逻辑 vs I/O）。

  • 芯粒式 WSI：利用晶圆级基板紧密集成、互连多个预先验证的芯粒。典型技术包括 Si-IF 和 台积电的 InFO-SoW。这种方式可以选择“已知良品芯粒 (KGD)” 集成到晶圆基板上，良率极高（>99.9%）。同时，可以将不同工艺制程的芯粒整合到同一系统中，例如 Tesla Dojo 或 Waferscale-GPU ，从而实现单片式 WSI 无法做到的架构。

文章采用芯粒式 WSI。假设所有芯粒与相邻芯粒以类似网格的方式互连。对于非相邻芯粒，通信通过多跳方式实现，中间芯粒作为中继。根据不同的集成工艺（见表 I），片内互连的功耗和带宽密度会有所差异。

而对于外部连接，通常使用晶圆边缘放置负责外部 I/O 的芯粒（Periphery I/O）。这些 I/O 芯粒在现有系统中大多基于 SerDes，但也可以采用 光学 I/O，即直接支持光收发器。两种方案中，I/O 芯粒都通过片上传输线与子交换芯粒 (SSC) 互连，如图 3.a 和 3.b 所示。只有位于晶圆边缘的芯粒能与 I/O 芯粒直接连接；若内部芯粒需要外部 I/O，则必须经由中继转发。

另一种方式是 面积 I/O (Area I/O)，其特点是 I/O 信号可以直接连接晶圆上任意芯粒（而不仅仅是边缘芯粒）。Area I/O 通过 贯穿硅通孔 (TWVs) 将信号和电力传导至晶圆另一侧，然后通过承载晶圆的 中介 PCB (Mezzanine PCB) 将这些信号重新分配到外部 I/O 模块。图 3.c 展示了 Area I/O 的剖面示意图。

1.2 构建晶圆级网络交换机

文章的物理实现是：在晶圆上构建一个由子交换机 (SSC) 组成的网格，每个 SSC 对应一个先进的单芯片交换机。采用网格布局的好处是能将长互连拆分为更短的段，提升能效、降低延迟并提高良率。目前已有的商业 WSI 系统（如 Cerebras WSE, Tesla Dojo）也采用物理网格结构。

在大部分分析中，文章使用 Broadcom Tomahawk 5 (TH-5) 作为子交换芯粒（SSC）的原型。TH-5 是一款 256 端口、200Gbps 的先进交换芯片。文章做了两点假设性修改：将 TH-5 芯片边缘的高速 SerDes 替换为更细间距的芯粒间 I/O。这样 SSC 可以通过细间距布线与四个相邻芯粒互连。由于网格拓扑的饱和吞吐量和双向带宽有限、延迟较高，不适合直接用作网络交换机，因此文章需要在物理网格上映射更理想的逻辑拓扑（如 Clos 拓扑），以获得高吞吐量和无阻塞特性。

为支持这种映射，文章假设 TH-5 的一小部分芯片面积（<2%）用于 贯通通道 (feedthrough channels)，以便信号跨越多个芯粒中继，从而在逻辑上实现长距离连接。例如，两个不相邻的 SSC 之间的逻辑连接，可以通过中间 SSC 转发来实现。在这种结构中：

  • 相邻 SSC 的延迟为 1ns。

  • 远端 SSC 的最坏情况延迟为 2N ns，其中 N² 为晶圆上 SSC 的总数。

  • 每个 SSC 的功耗约为 500W（去掉 I/O 功耗后为 400W），面积约 800mm²。

图 4 展示了 Clos 拓扑如何映射到物理网格上的一个例子。部分 SSC 之间直接相连，而另一些则通过中间 SSC 转发。这样，即使逻辑上需要全互连，物理网格依然能支持。

2. 量化晶圆级网络交换机的端口数（Radix）优势

2.1 映射 Clos 拓扑

如何将逻辑 Clos 拓扑映射到物理网格上，将直接决定晶圆级网络交换机的性能表现。映射的核心目标是 最大化交换机的端口数 (Radix) 以及 双向带宽 (Bisection Bandwidth)。作者采用了成对交换 Pairwise Exchange的启发式算法。

实验结果（图 5）显示，该算法相比随机映射能使 最坏情况下的片内 I/O 带宽/端口提升约 147.6%，说明映射优化能显著改善互连效率。

2.2 理想与现实情况对比

仅考虑晶圆面积约束

在仅考虑晶圆面积约束、忽略功耗与带宽瓶颈时，利用上述映射方法，文章可计算出晶圆级交换机的最大可支持端口数（图 6）。

假设晶圆为正方形，例如边长 100mm 对应面积为 100×100 mm^2。TH-5 的三种典型配置（见表 II）在图中以红色方块标出。结果表明：

  • 在 300mm 晶圆 上，基于芯粒封装的 WSI 可以实现比单个 TH-5 高出 32 倍 的端口数。
  • 在 200mm 晶圆 上提升 16 倍，在 100mm 晶圆 上提升 4 倍。

这种数量级的提升意味着网络拓扑可大幅减少跳数与延迟，甚至可实现大规模节点间的直接互连。不过也观察到，若在相同晶圆面积下提升单端口带宽（例如从 200G 提升到 800G），最大可支持的端口数会下降。但即使在高带宽场景中，晶圆级方案仍可带来 2–8 倍 的端口数增益。

考虑带宽瓶颈

在理想情况下忽略的带宽瓶颈，实际中会显著限制端口扩展。图 7 展示了在内部互连带宽密度为 3200 Gbps/mm（对应 Si-IF 技术）的条件下，不同外部 I/O 技术所能实现的最大端口数。

  • 光学 I/O (Optical I/O)：3200 Gbps/mm
  • 面积 I/O (Area I/O)：16 Gbps/mm²

产生这种差距的原因是：

  2. 当内部带宽达到饱和（例如在 200mm 晶圆时），即便增大晶圆尺寸（至 300mm），端口数也不会再提升。
  3. 当将内部带宽翻倍至 6400 Gbps/mm（图 9），光学 I/O 下的端口数可增加 2–4 倍，接近理想值。
  4. 面积 I/O 的提升有限，因为其带宽密度天生低于光学 I/O。

考虑功耗与散热约束

上述分析未考虑功耗和散热限制。图 10 显示，在 200mm 与 300mm 晶圆下，当采用光学 I/O 或面积 I/O 时，总功耗超过 14 kW。在实际系统中，考虑到冷却方式（空气冷却、水冷或多相冷却），可承受的功率密度有限，因此最大可支持的端口数还将进一步下降。

晶圆级交换机在理论上可实现惊人的扩展性，但其潜力受限于三大现实瓶颈：

  • 内部互连带宽（SSC 之间的通信受限）

  • 外部 I/O 带宽（传统 SerDes 无法匹配晶圆规模）

  • 功率密度与散热能力

3. 可扩展的晶圆级网络交换机 SCALABLE WAFERSCALE NETWORK SWITCHES

3.1 通过牺牲能效提升 I/O 带宽

存在一些晶圆级集成（WSI）技术能够在提高功耗的前提下提供更高的 I/O 带宽。
例如，台积电的 InFO-SoW 技术可实现约 4 倍带宽密度（见表 I），但其能效较低（1.5 pJ/bit，相比 Si-IF 的 0.06 pJ/bit）。类似地，Si-IF 技术也可以通过提高链路频率和电压来提升带宽，但代价是更高的能耗。

文章的目标是在合理的功率预算内最大化交换机端口数 (radix)。
因此，文章设定目标为将内部 I/O 带宽密度从 3200 Gbps/mm 提升到 6400 Gbps/mm（即每层 1600 Gbps/mm，总 4 层），通过提高工作频率并相应提升供电电压 。

  • 在 100mm 晶圆 上变化较小，因为此时光学 I/O 已接近理想上限。

  • InFO-SoW 的最大端口数与 6400 Gbps/mm 的 Si-IF 相当。
  • 但总功耗高达 92.5 kW。

因此，后续分析采用 Si-IF 作为主要 WSI 技术。

3.2 异构网络交换机设计

前述分析假设晶圆上所有 SSC（子交换芯片）均相同，形成同构 Clos 拓扑。 本节提出一种新的 异构设计。

Clos 的特性允许将叶节点 (Leaf SSC) 拆分为多个较小芯片，只要与上层 主干节点 (Spine SSC) 的连接保持不变，系统总体端口数不会减少。例如，一个 256 端口的叶子 SSC 可替换为两个 128 端口的小芯片。这样做会略微增加延迟（约 1%），但可显著降低功耗。

实测表明，多款商用高端口交换机（如 Broadcom Tomahawk 系列与 Marvell Teralynx 系列）的功耗随端口数呈超线性（二次）增长。图 15 展示了这一趋势：功耗几乎随端口数平方增长，这意味着两个小交换机的功耗总和明显低于一个同等端口数的大交换机。因此，使用较小端口数的 SSC 替换部分高端口 SSC，可有效降低系统功率密度。

图 16 展示了异构优化的结果：

  • 在 300mm 晶圆 上，采用小型 TH-3 级芯粒作为叶节点，可使总功耗降低 30.8%。
  • 功率密度从 0.69 kW/mm² 降至 0.48 kW/mm²，适合 水冷系统 散热（根据 ，水冷可支持约 0.5 kW/mm² 热密度）。

注意：此优化主要降低了 SSC 内部逻辑功耗，对内部 I/O 功耗影响较小。因此，随着系统规模增加，功率下降比例减小，因为 I/O 功耗占比上升。

3.3 子交换机降阶（Subswitch Deradixing）

另一种“反直觉”的优化方法是：降低每个 SSC 的端口数（radix），但保持芯片面积不变。
这样能为每个芯片分配更多的“穿通 I/O”（feedthrough I/O），增加芯片间互连带宽。

原理如下：

  • 每个 SSC 的 I/O 分为两类：对外端口 I/O 和内部互连 I/O。
  • 当减少外部端口时，内部互连资源增加，可连接更多芯粒。
  • 结果：整体系统端口数反而上升。

图 19 显示这一趋势（300mm 晶圆为例）：

  • 当 SSC 为 256 端口时，系统可配置 8192 或 4096 端口，但单端口带宽不足 200 Gbps。
  • 只能运行在 2048 端口模式下。
  • 若将每个 SSC 降为 128 端口，则每端口带宽充足，系统整体端口数反而可达 4096。

图 17 和图 18 进一步展示：

  • 在 3200 Gbps/mm 下，将 SSC 端口数减半可使系统端口数翻倍。
  • 在 6400 Gbps/mm 下，该趋势仍然存在，但当带宽充足时，过度降阶会导致端口利用率下降。

为突破第四章识别的瓶颈，本章提出三类可扩展性优化策略：

  1. 提高内部带宽密度（通过频率、电压提升）以扩展端口数，但需权衡功耗。
  2. 异构网络交换机设计，通过组合不同端口数芯粒，降低整体功率密度（减少约 30%）。
  3. 子交换机降阶（Deradixing），通过减少单芯片端口数换取更多晶圆级互连，从而增加整体系统规模。

这些策略共同作用，使晶圆级网络交换机在可扩展性、性能与能效之间达到更优平衡。

仿真配置如下：

  • 路由器具有 64 个虚拟信道 (VCs)
  • 等效延迟为 200ns
  • 所有输入端口共享一个缓冲区 (Shared Buffer)
  • 每个仿真周期定义为 20ns（200ns = 10 个仿真周期）
  • flit 长度调整为与 TH-5 的线路速率匹配

较小的缓冲不仅节省芯片面积与功耗，还能降低访问延迟，因为小缓冲可用高速 SRAM 实现，而大缓冲通常需用较慢的 DRAM 。因此，晶圆级交换机上的 SSC 相比传统商品交换机（如 TH-5）具有更低的延迟，主要得益于缓冲需求的减少。

4.1 定制低延迟路由机制

晶圆级网络交换机还可通过 定制低延迟路由逻辑 进一步提升性能。即通过晶圆级网络的拓扑在设计时已固定的前提，在 非入口 SSC（non-ingress SSC） 中完全跳过 IP 查表。

在传统三层路由（Layer-3 Routing）中，交换机需进行 最长前缀匹配 (Longest Prefix Match) 来确定目的地址，这会增加 IP 查表延迟。

具体机制如下：

  • 在 入口 SSC (Ingress) 执行正常的 IP 查找，计算出整个交换机的目标端口；
  • 将目标端口信息添加到数据包头部；
  • 主干 (Spine SSC) 和 出口 (Egress SSC) 不再进行路由查表，而是直接根据目标端口进行转发；
  • 出口交换机在发包前删除额外的头部信息。

这种方法可消除多级查找延迟。估算显示，不含 IP 查表的路由计算（RC）延迟仅为传统的 1/4。

4.2 仿真评估：专用路由的性能收益

研究者使用 Booksim2 对比了带 IP 查表与不带 IP 查表的性能差异。
仿真设置如下：

  • 网络结构：两级 Clos 网络
  • 组件：96 个 radix-256 的 SSC，总端口数 8192
  • 交换延迟：16 个仿真周期
  • 路由延迟（基线）：4 个仿真周期
  • 每端口缓冲区：128 flit
  • 虚拟信道数：64
  • 不使用内部速率加速（No internal speedup）
  • 流量模式：随机均匀 (Random Uniform)

优化配置中：

  • 入口 SSC 路由延迟设为 2 个仿真周期
  • 非入口 SSC 路由延迟设为 1 个仿真周期

结果（图 22）表明：

  • 使用专用路由后，零负载延迟 (Zero-load latency) 明显降低；
  • 饱和吞吐量 (Saturation Throughput) 提升显著：
    • 200mm 晶圆交换机提升 14.5%
    • 300mm 晶圆交换机提升 11%

4.3 应用层性能评估

为估计晶圆级交换机在应用层的潜在性能收益（包括更快互连和更快 SSC 微架构），
研究者对比了以下两种系统的性能：

  • 2048 端口、800 Gbps 的晶圆级交换机
  • 等效的传统多层 TH-5 网络

仿真参数如下：

  • 虚拟信道数：16
  • 每输入端口缓冲区：32 flit
  • SSC 延迟：11 个仿真周期
  • 基线交换箱延迟：15 个仿真周期
  • 主机到入口交换机 I/O 延迟：8 个仿真周期（两者相同）
  • 晶圆级交换机中 SSC 之间延迟：1 个仿真周期
  • 传统网络中交换机之间延迟：8 个仿真周期

结果（图 23）：

  • 晶圆级交换机在所有流量模式下（除非对称流量外）具有相同或更高的饱和吞吐量；
  • 零负载延迟降低 38%，从 60 个仿真周期降至 37 个仿真周期。
