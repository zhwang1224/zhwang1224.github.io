DOJO super-compute system scaling for ML training

1. System Architecture

Tesla 实现“通用的自动驾驶能力”，需要三个支柱：模型、数据、训练基础设施。模型从视觉感知到路径规划到自动标注，模型越来越复杂，参数量爆炸；数据需要使用视频训练数据；训练基础设施需要高效训练以及评估。

典型系统在计算、输入输出以及存储方面是固定比例，DOJO 把计算、内存、I/O 分拆成独立可扩展的资源池，用高速互联把它们连起来；

这样可以按训练任务的需要去“配料”：

  • 大模型 → 多内存 / 多带宽；
  • 预处理重 → 多 CPU + I/O 资源；
  • 算力密集 → 多 Tile / 多 Dojo 芯片。

这就是 Dojo 整体设计思想之一：资源解耦 + 高带宽互联。

2. Optimized Compute

晶圆技术和封装技术驱动计算芯片规模扩展

  1. 晶圆级系统集成技术（System-On-Wafer Technology）

    • 25 （5x5 mesh） 颗 D1 计算芯片 + 40 颗 I/O 芯片

    • 计算芯片与 I/O 芯片的分工设计，用于优化效率与互联覆盖范围

    • 异构 RDL（再布线层），针对高密度、高功耗布局进行优化

  2. 性能与良率最大化（Maximize Performance and Yield）

    • 使用“已知良好芯片”（Known Good Die）和容错设计

    • 每个 Tile 由完全功能正常的芯片组装而成

    • 通过“收割（Harvesting）”和完全可配置的路由来提升良率

Training Tile

Tesla 将这样的晶圆级芯片作为扩展的基本单元，具有大规模计算能力，配套优化的 I/O，以及完全集成的系统模块（含电源 / 散热）。Uniform 下 Tile 内部对分带宽：10 TB/s；Tile 与外部之间的总带宽：36 TB/s。性能与存储规格方面，9 PFLOPS（每秒 9×10¹⁵ 次）BF16/CFP8 计算性能，11 GB 高速 ECC SRAM和 36 TB/s 的总 I/O 带宽。

可以灵活地构建模块（Flexible Building Block），每条边可以提供 9 TB/s 的带宽。通过多个 Tile 进行扩展，扩展时无需重新设计电源 / 散热系统。

3. Disaggregated Memory & IO

在 Dojo 里，Tile 自身主要提供算力 + 高速 SRAM，而大容量 DRAM 由其他模块（如 DIP+HBM）提供。V1 Dojo Interface Processor（Dojo 接口处理器 V1）。

  1. 32 GB 高带宽内存（HBM）
  2. 900 GB/s 的 TTP 接口 为训练 Tile 提供全部 DRAM 带宽
  3. 50 GB/s 的 TTP-over-Ethernet（TTPoE） 通过标准以太网扩展 TTP 通信
  4. 32 GB/s 的 PCIe Gen4 接口

每块 Tile 的一条边上布置若干 DIP+HBM 卡，每个 DIP 带 32 GB HBM，5 张 DIP 卡合计约 160 GB DRAM，这 160 GB 作为该 Tile 的“共享内存池”。5 张 DIP 卡即通过 TTP 向 DRAM 提供 4.5 TB/s 的总带宽。80 条 PCIe Gen4 通道为主机提供标准的 PCIe 连接能力。

Dojo 接口处理器 —— Z 平面拓扑

TTPoE：基于以太网的点对点连接

  • 在 Z 平面（跨机柜 / 跨机架方向）提供高端口数的 TTP 网络连接

  • 通过以太网交换机建立网络中的“捷径”（shortcuts）

  • 用于管理计算平面上的同步与控制延迟

    • 第一种情况：不使用这些捷径时，某些路径可能需要约 30 跳（~30 hops）

    • 加入 TTPoE + 以太网交换机后，可以将关键路径缩短到约 4 跳（~4 hops）


Dojo 网络接口卡（DNIC：Dojo Network Interface Card）

DNIC部署在主机（Host）上，通过 TTPoE 实现远程 DMA（RDMA）：可以对任何 TTP 端点进行 DMA 读写（包括计算节点的 SRAM、DRAM 等），复用现有的以太网交换网络。


4. V1 Dojo Training Matrix

第一代 Dojo 训练系统”的全局规格：

  • 计算：达到 1 EFLOP 级别（在低精度 BF16/CFP8 下）；

  • On-chip SRAM：1.3 TB，作为训练时的主工作存储；

  • 外部高带宽 DRAM：13 TB，容纳模型参数和部分激活。

这是一个典型的“统一内存 + 统一算力池 + 统一网络”训练超级计算机。

  • 计算：Tile

  • 接口处理器：DIP（提供 HBM、TTP/TTPoE）

  • 网络接口：DNIC（连接到 CPU/DRAM、以太网）

共同组成：解耦且可按需扩展的计算 / 内存 / I/O 系统
